集合框架的概述

  1. 集合、数组都是对多个数据进行存储操作的结构，简称java容器
    • 说明:此时的存储，主要指得是内存内面上的存储，不涉及到持久化存储，比如（.txt, .jpg,数据库中的存储）
  2. 数组在存储多个数据的特点
    1. 一旦初始化后，长度就确定了，不可以改变
    2. 数组一旦定义好，元素类型也确定了，只能存储指定类型的数据

Java集合课分为Collection 和 Map 两种体系

  1. Collection接口：单列集合，用来存储一个一个的对象
    1. List:元素有序、可重复的集合 –> “ 动态数组 ”
      1. ArrayList、LinkedList、Voctor
    2. Set:元素无序、不可以重复 –> “ 数学中的集合 ”
      1. HashSet、LinkedHashSet、TreeSet
  2. Map接口：双列集合，用来存储一对一对的数据 “ key - value “ 键值对 –>”函数“
    1. ​ HashMap、LinkedHashMap、TreeMap、Hashtable、Properties

Collection接口中的方法使用

  1. size() :返回 集合中的元素个数
  2. add(Object obj) :向集合中添加数据
  3. addAll(collection c ) :将指定集合中的所有元素添加到当前集合中
  4. isEmpty( ): 判断集合是否有元素， 如果有返回 true 否则 返回 false
  5. clear( ): 清空集合元素

  1. contains(Object obj )判断 obj 是否包含在集合中，如果包含返回True，不包含返回 false

    1. containsAll( Collection coll )判断集合coll 是否包含在当前集合中

    • contains（）方法是使用 equals()来进行判断的

  1. remove(Object obj):从当前集合中移除obj元素

  2. removeAll(ection coll)：从当前集合中移除coll集合中的所有元素

  1. retainAll(Collection coll)返回当前集合和coll集合中的相同元素给当前集合数学中的交集

  2. equals(Object boj): 判断当前集合和 指定的obj集合的元素是否相同，如果相同返回true 否则返回false

  3. hashCode( ): 返回当前对象的哈希值

  4. toArray( ): 把集合转换为数组

  5. Arrays.asList( ): 把数组转换为集合

    1. iterator( ): 返回当前集合的迭代器

List 接口

  1. List 集合中的元素有序，并且可以重复，集合中的元素都有与其对应的顺序索引

  2. List接口的实现类有：ArrayList、IinkedList、Vector

  3. ArrayList、IinkedList、Vector、三者的异同

    • 相同点：都实现了List接口，存储数据的特点相同，都是有序的，可以重复的

    • 不同点：

      ArrayList：作为List接口的主要实现类，线程不安全，效率高，底层是使用Object[] elementDate存储

      LinkedList：对于频繁的插入，删除操作，使用此类效率比ArrayList效率要高；底层使用的是链表的方式存储数据。

      Vector：作为List接口的古老实现类；线程安全，效率低，底层使用Object[] elementDate存储

Map ：双列数据，存储key - value对的数据

  1. HashMap: Map 的主要实现类，线程不安全，效率高，可以存储 null 的 可以 - value
    • LinkedHashMap: HashMap 的子类，保证遍历元素时可以按照添加时的顺序实现遍历
      • 原理：在原有的HashMap上添加一对指针，指向前一个元素和后一个元素。对于频繁的遍历操作，LinkedHashMap执行效率比HashMap的执行效率要高
